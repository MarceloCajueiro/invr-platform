# Phase 2.5: AI Integration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** AI-powered content generation (quiz, fill-gaps, writing prompts via Groq), writing correction (auto-grade via Groq), and TTS audio generation (Gemini Flash) for listening exercises.

**Architecture:** AI services in `src/lib/services/ai/`, Server Actions for generation flows, Route Handlers for binary data (TTS). Groq SDK for LLM, Google Generative AI SDK for TTS. Rate limiting tracked per teacher.

**Tech Stack:** Groq SDK, @google/generative-ai, Server Actions, Route Handlers

---

### Task 1: AI Service Layer (Groq + Gemini)

**Files:**
- Create: `src/lib/services/ai/groq.ts`
- Create: `src/lib/services/ai/gemini.ts`

**groq.ts:** Groq client wrapper with functions:
- `generateQuizQuestions(prompt, level, count)` — system prompt requesting JSON array of quiz questions
- `generateFillGapQuestions(prompt, level, count)` — same for fill-gaps
- `generateWritingPrompt(theme, level)` — generates prompt + rubric
- `correctWriting(text, prompt, level)` — returns score, feedback, errors, tips

**gemini.ts:** Gemini Flash client wrapper with:
- `generateTTS(text, speed?)` — text-to-speech, returns audio buffer

- [ ] Install groq-sdk and @google/generative-ai
- [ ] Create Groq service with all generation functions
- [ ] Create Gemini TTS service
- [ ] Commit
